# The Whales

The Whales는 조작 중심 게임이 아니라, 고래들이 살아가는 흐름을 관찰하는 **상태 기반 생태계 경험**을 목표로 합니다.

---

## 프로젝트 방향 (업데이트)

기존 개념:
- 고래 = AI agent
- 상호작용 = 결과 생성(음악/결과물)

현재 개념:
- 고래 = 상태 기반 존재
- 행동 = 목표 수행이 아니라 상태 반응
- 상호작용 = 결과물이 아니라 상태 변화의 표현

핵심 원칙:
> 고래는 무언가를 하기 위해 만나지 않는다.  
> 서로의 상태에 반응하기 때문에 만난다.

---

## 현재 행동 모델

각 고래는 다음 값을 가집니다.

- `state` (예: calm, active, wandering, drifting, curious)
- `energy`
- `nearbyDensity` (주변 밀도 인식)

틱/프레임마다:
1. 주변 고래 인식
2. 자신의 상태 + 주변 상태 비교
3. 확률 기반 행동 선택

행동 종류:
- `approach`
- `align`
- `circle`
- `drift_away`
- `ignore`

중요 규칙:
- 동일 상황에서 동일 행동 반복 금지
- 확률 기반 다양성 유지
- “자연스럽게 발생한 것처럼” 보여야 함

---

## 상호작용 정의

상호작용은 이벤트 결과물이 아니라 **표현**입니다.

표현 요소:
- 움직임 변화 (같이 흐름, 방향 정렬)
- 거리 변화 (가까워짐/멀어짐)
- 짧은 소리
- 미묘한 시각 효과

금지:
- 강한 UI 텍스트 피드백
- 명시적 이벤트 알림
- “상호작용 발생” 같은 직설적 표시

사용자는 직접 제어하지 않고 “알아채는 경험”을 해야 합니다.

---

## 유저 경험 원칙

유저는:
- 고래를 조종하지 않음
- 행동을 명령하지 않음

유저가 하는 것:
- 특정 고래를 따라감
- 고래 사이의 반응을 관찰함
- 흐름에 애착을 형성함

---

## 지금 당장 제거/리팩토링 대상

다음 구조는 유지하지 않습니다.

- 목표 기반 행동 (`goal`, `task`, `objective`)
- 의도 중심 의사결정 트리
- 음악/결과물 생성이 목적이 되는 구조
- 유저에게 생산 결과를 전달하려는 파이프라인

프로젝트 정체성:
> 생산 시스템이 아니라 살아있는 생태계

---

## AI agent 관련 현재 정책 (중요)

현재 환경에서는 고래를 “실시간 LLM 기반 AI agent”로 운영하지 않습니다.

이유:
- 실시간 추론 비용/지연
- 플랫폼 운영 복잡도
- 결정성/재현성/성능 안정성 요구

대신 현재는:
- 상태 기반 생태계 모델을 기본 런타임으로 채택

향후 확장 여지:
- 특정 고래/희귀 개체에 제한적 AI layer 연결
- 서버 보조형 추론(비동기)과 로컬 상태 시스템 하이브리드
- 시즌 이벤트성 고래 행동 확장

즉, **지금은 상태 기반**, **미래에는 AI agent 확장 가능**으로 설계합니다.

## 현재 성공 기준

아래가 충족되면 성공입니다.

- 플레이어가 없어도 세계가 계속 살아있는 느낌
- 고래들이 서로 의미 있는 존재처럼 보임
- 유저가 특정 고래에 감정적으로 끌림
